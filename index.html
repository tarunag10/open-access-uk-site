Suite roadmap

A lightweight release board for the seven repos, grouped by what is publishable now, what should land next, and what belongs in later slices.

Now

  • Open Access UK site: publishable front door with repo status, contribution routes, GitHub links, and suite verification notes.
  • Reasonable Adjustment Letter Generator: local drafting flow with copy, print, and `.txt` export.
  • Accessible Public Forms: static form examples with readiness checks and role-friendly contribution guidance.
  • Public Service Directory: sourceable escalation-route data and local action-plan generation.
  • Legal Templates UK: searchable template catalogue with copy and `.txt` export.
  • Open Access Design System: tokens, component examples, copy patterns, and snippet output.
  • Open Source Maintainer Helper: readiness scoring and grouped good-first issue suggestions.

Next

  • Open Access UK site: add release notes, issue-label conventions, and hosted demo links once public pages are enabled.
  • Reasonable Adjustment Letter Generator: broaden organisation-specific wording and source-review metadata.
  • Accessible Public Forms: add more complete public-service journeys and validation examples.
  • Public Service Directory: expand official-source coverage and add printable/exportable action plans.
  • Legal Templates UK: add review status labels, source notes, and deadline caveats per template.
  • Open Access Design System: document component acceptance checks and usage guidance.
  • Open Source Maintainer Helper: add exportable issue drafts and clearer repository health scoring.

Later

  • Open Access UK site: aggregate live GitHub issue metadata after privacy and maintenance review.
  • Reasonable Adjustment Letter Generator: optional saved drafts if a local-only storage model is approved.
  • Accessible Public Forms: package reusable form specs for other public-interest projects.
  • Public Service Directory: add community-reviewed route update workflow.
  • Legal Templates UK: build a contributor review queue for jurisdiction and source checks.
  • Open Access Design System: publish versioned design tokens for downstream repos.
  • Open Source Maintainer Helper: support import/export of repository audit snapshots.

How the suite should behave

The recommendation brief points toward practical public-interest tools that are safe to inspect, easy to fork, and honest about their limits.

Keep it local

Prefer static HTML, CSS, and JavaScript. Do not add a backend, analytics, or personal-data collection without a public privacy review.

Design for access needs first

Use semantic HTML, visible focus states, clear labels, plain language, high contrast colours, and keyboard-only testing before visual polish.

Show sources and limits

Directory entries and templates should be sourceable. Drafting tools must avoid claiming to decide rights, outcomes, or legal strategy.
