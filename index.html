Sources and safety

Source-backed, honest about limits.

High-risk content is reviewed against public sources and marked as information, not legal advice. If something looks stale, unclear, or unsafe, report it through GitHub or the support route.

Source records

Every high-risk tool has source notes, review dates, and a reviewer in the parent data registry.

Safety language

Tools explain that templates and routes are drafting support, not legal advice.

Report stale content

Use GitHub issues or support docs to flag broken links, outdated guidance, or unsafe wording.

Privacy centre

Local-first means you stay in control.

The tools do not need accounts, analytics, or server storage. Some tools can save drafts, favourites, notes, or plans in your browser so you can continue locally.

Known local data

Drafts, saved plans, favourites, review notes, and component shortlists stay in browser storage.

Clear locally

Each tool exposes clear/reset controls where it saves local data.

No hidden collection

Analytics are off by default and generated letters, templates, and reports are not sent to a server.
